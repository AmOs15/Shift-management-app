# シフト希望入力モックアプリ

従業員がログインし、カレンダーからシフト希望時間を入力・確認できる Next.js 製のモックアプリです。

UI、画面遷移、操作感の確認を目的としており、データベース、API、本番用認証、複数ユーザー管理、管理者機能は実装していません。

## 使用技術

- Next.js
- App Router
- TypeScript
- Tailwind CSS
- `@daypicker/react`
- React Context / custom hooks
- `sessionStorage`
- npm

## セットアップ

```bash
npm install
```

カレンダーライブラリを追加する場合のコマンド例です。

```bash
npm install @daypicker/react
```

## 開発サーバー

```bash
npm run dev
```

起動後、ブラウザで `http://localhost:3000` を開きます。

## ビルドと確認

```bash
npm run lint
npm run build
```

## データ保持

認証状態とシフト希望データはブラウザの `sessionStorage` に保存されます。

- ページ遷移後もデータは残ります。
- 同じタブで再読み込みしてもデータは残ります。
- 新しいタブとはデータを共有しません。
- タブまたはブラウザを閉じるとデータは消えます。
- ログアウトしてもシフト希望データは同じタブ内に残ります。
- メニュー画面の「データをリセット」からシフト希望データのみ削除できます。

## 注意事項

このアプリの認証はモックです。氏名とパスワードはフロントエンドのコード上に定義され、ブラウザへ配信される JavaScript から確認できます。

本番利用できるセキュリティ機構ではありません。

## Vercel デプロイ

Vercel へ Next.js アプリとしてデプロイできます。

- 環境変数は不要です。
- データベースは不要です。
- API サーバーは不要です。
- ファイルシステムへの書き込みは行いません。

通常の Vercel 連携でリポジトリを選択し、プロジェクトルートを `frontend` に設定してデプロイします。
